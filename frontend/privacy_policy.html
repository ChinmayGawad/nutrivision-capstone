Legal Compliance Document

🔒 Privacy Policy & Data Protection

NutriVision AI — Food Nutrition Estimator

Last updated: March 2026

1. Overview

NutriVision AI is committed to protecting user privacy in compliance with applicable data protection laws including India's Digital Personal Data Protection (DPDP) Act, 2023 and the EU's General Data Protection Regulation (GDPR).

2. Data We Process

  • Food Images: Uploaded images are processed entirely in-memory for AI-based food recognition. Images are never written to disk, stored in a database, or transmitted to third parties.
  • Nutrition Queries: When our local database cannot identify a food item, the food name (text only — not the image) may be sent to the Edamam Nutrition API for lookup.
  • IP Addresses: Logged temporarily for rate limiting and security audit purposes as permitted under legitimate interest.

3. Data Minimization (DPDP Act Section 4 & GDPR Article 5)

We implement data minimization by design: only the minimum personal data required for the service (food image for recognition) is processed. Images exist only in server RAM during processing and are immediately discarded. No persistent storage of personal data occurs.

4. Legal Basis for Processing

  • Consent (DPDP Act Section 6): By uploading an image, the user consents to its processing for nutrition estimation.
  • Legitimate Interest (GDPR Article 6(1)(f)): Security logging (IP addresses, failed authentication attempts) is maintained for system security and fraud prevention.

5. User Rights

Under DPDP Act 2023:

  • Right to access information about personal data processing (Section 11)
  • Right to correction and erasure of personal data (Section 12)
  • Right to grievance redressal (Section 13)
  • Right to nominate (Section 14)

Under GDPR:

  • Right to Access (Article 15), Rectification (Article 16), Erasure (Article 17)
  • Right to Restriction of Processing (Article 18)
  • Right to Data Portability (Article 20)

6. Security Measures

  • JWT-based authentication with HS256 signing and automatic token expiry
  • Role-Based Access Control (RBAC) for administrative endpoints
  • Rate limiting (DDoS protection) via sliding-window algorithms
  • Input validation: MIME type checking, file size limits (5MB)
  • HTTP security headers: HSTS, CSP, X-Frame-Options, X-Content-Type-Options
  • CORS policy restricting API access to authorized frontends
  • Environment variable isolation for secrets (never committed to source code)
  • Centralized audit logging for forensic investigation

7. Third-Party Services

  • Edamam API: Used as a fallback nutrition data source. Only food name text is sent (not images). See: Edamam Privacy Policy

8. Data Retention

Uploaded images: 0 seconds (processed in-memory, immediately discarded).
Security audit logs: Retained for 90 days for incident response purposes, then purged.
JWT tokens: Auto-expire after 1 hour and are not stored server-side (stateless).
